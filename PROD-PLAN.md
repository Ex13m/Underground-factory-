# UNDERGROUND FACTORY — план боевой версии

Анализ прототипа и план проектирования продакшна. Статус: НА СОГЛАСОВАНИИ у владельца
(по процессу spec-pilot: сборка — только после явного «да»).

## 1. Что имеем (аудит прототипа)

**Сильное (актив, который нельзя потерять):**
- Уникальный фронт: дизайн-система техвир, FX, бот-продавец, видео-витрина,
  арт-конвейер (генерации, библиотека тачек, авто-заставки) — этого нет ни в одной теме Shopify.
- Слой данных `UF_API` уже спроектирован как контракт с адаптерами (API.md):
  бэкенд подключается заменой адаптеров, интерфейс не меняется.
- Контент-завод: Higgsfield/Replicate конвейер, рилсы, радио — готовый маркетинг-цех.

**Слабое (что блокирует продажи по-настоящему):**
- Все данные в браузере (localStorage/IndexedDB): нет синхронизации между устройствами,
  нет реальных заказов, «оплата» — заглушка.
- Открытые серверные функции: /api/queue и /api/track принимают POST от кого угодно —
  в проде это дыра (спам, залив файлов).
- Админка на клиентском коде 4174 — защита «от честных людей».
- SPA без SSR: поисковики видят пустую страницу — SEO почти нулевое.
- Каталог: ~12-50 SKU, до 8-10 тачек фитмента — МАЛЕНЬКИЙ. Это важно для выбора платформы.

## 2. Shopify или своя структура?

### Вариант A — Shopify (headless, Storefront API)
Наш фронт остаётся, Shopify берёт каталог, корзину, чекаут, платежи, налоги.
- **Реально ли**: да. Storefront API + наш React — стандартная схема; ~1-2 недели интеграции.
  «Buy Button»-путь ещё проще, но уродует UX.
- **Плюсы**: платежи/налоги/PCI/фрод — их проблема; админ-панель заказов из коробки;
  надёжность.
- **Минусы**: от $39/мес + комиссия с транзакций; каталог и промо-логика переезжают
  в их модель (наши промокоды бота, гараж, фитмент-матрица тачек — придётся дублировать
  метаполями); зависимость от платформы; наш арт-конвейер всё равно остаётся отдельно.
- **Когда оправдан**: если нужны быстро налоги ЕС/US, доставка-фулфилмент, объём >500 заказов/мес.

### Вариант B — своя структура (РЕКОМЕНДАЦИЯ)
Тонкий бэкенд на том, что уже есть: Netlify Functions + управляемый Postgres
(Supabase — заодно даёт Auth и Storage) + **Stripe Checkout** (hosted-страница оплаты).
- **Почему для нас**: каталог маленький и нестандартный (фитмент тачки↔детали, грейды
  материалов, промо-эскалация бота) — в Shopify это костыли, у нас уже смоделировано;
  `UF_API` подключается заменой адаптеров; комиссия только Stripe (~2.9%+30¢), без
  платы за платформу; полный контроль над UX.
- **PCI**: Stripe Checkout = карты не касаются нашего кода (SAQ-A, минимальный уровень).
- **Минусы**: заказы/почта/налоги собираем сами (для старта: Stripe Tax + вебхуки + письма
  через Resend — решаемо); ответственность за бэкапы и аптайм.
- **Объём работ**: ~3-4 недели до боевого магазина (этапы ниже).

**Вывод**: при «не миллионе деталей» Shopify не даёт главного — он не умеет наш фронт
и нашу доменную модель, а его сильные стороны (масштаб, логистика) нам пока не нужны.
Своя структура на Supabase+Stripe — дешевле, гибче и использует всё уже построенное.
Shopify держим как план Б на случай выхода на объёмы/международные налоги.

## 4. Безопасность (чек-лист прода)

- Секреты только в env (Netlify), ключи не в клиенте; REPLICATE_API_TOKEN — серверный.
- /api/* : авторизация (JWT Supabase), rate-limit, валидация схем (zod), CAPTCHA на формах.
- Оплата только через Stripe-hosted (PCI SAQ-A), вебхуки с проверкой подписи.
- Админка: настоящий логин + 2FA, аудит-лог действий, разделение ролей.
- БД: бэкапы (Supabase daily), RLS-политики на таблицы.
- Заголовки: HSTS, CSP (запрет inline-скриптов кроме vite), X-Frame-Options.
- Персональные данные: политика конфиденциальности, cookie-баннер уже есть, право на удаление.
- Медиа-загрузки: лимиты размера/типа, антивирус-скан не нужен для картинок, но MIME-проверка обязательна.

## 5. Маркетинг (что включаем)

- **Контент-мотор** (уже есть): рилсы-завод + автопостинг → соцсети; видеосалон как витрина.
- **SEO**: карточки деталей с фитментом («сплиттер на Silvia S15») — низкочастотное золото.
- **Email**: подтверждения, брошенные корзины, дропы новых обвесов (стиль «сводок цеха»).
- **Промо-механики**: эскалация бота (есть), сезонные дропы, реферальный код гаража.
- **Аналитика**: Plausible (без cookie-головняка) + события: корзина, промо, бот.
- **Соцдоказательство**: фото клиентских тачек (UGC-гараж) с модерацией в админке.
